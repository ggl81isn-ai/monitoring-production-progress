# 信号以外の進捗表現（実装前メモ）

週次スクショ用に「一目で状態が伝わる」候補。Notionの「制作」タスクや日付が取れるようになってから、組み合わせを選ぶとよい。

## すぐ効きやすいもの

1. **週次ステータス1語 + 色帯**  
   例：「順調」「注意」「要リカバリ」をカード上部の細い帯（ティール／琥珀／スレート）で表示。信号と同じ判定を別表現するだけでも伝わりやすい。

2. **7日ストリップ（活動ドット）**  
   その週の月〜日の7マス。制作タスクに更新があった日だけ塗る。LINE画像でも「今週動いたか」が一瞬でわかる。

3. **今週の数値3つまで（ミニKPI）**  
   例：完了数、着手数、ブロッカー件数。大きな数字1つ＋小さく先週比。信号の「なぜ赤か」とセットで説得力が出る。

4. **全体進捗バー（フェーズ比率）**  
   `data-current-step` を `(n-1)/(N-1)` に換算し、細いプログレスバー1本。ロードマップの補助線として有効。

## 中期的に意味が出やすいもの

5. **バーンダウン風「残り工数目安」**  
   見積もりストーリーポイントや残タスク数が取れるなら、残り本数＋予測線はモチベーション可視化に向く。

6. **ブロッカー／リスクチップ**  
   「なし」「1件：レビュー待ち」のように最大2行。赤信号の理由を構造化できる。

7. **次マイルストーンまでのチェックリスト**  
   次の1つのマイルストーンだけに絞り、完了条件を2〜4チェックで表示。全体ロードマップと役割分担できる。

## 組み合わせの例

- **最小**: 信号 + キャプション1行（現状に近い）  
- **推奨（ややリッチ）**: 信号 + 7日ストリップ + 今週数値1つ  
- **長期トラック重視**: コンパクトロードマップ + 全体バー + 週次ステータス帯  

実装する際は、スクショの縦長を増やしすぎないよう **1画面に収まる上限** を先に決めるとよい。

# GitHub Actions の Artifact（成果物）の見方

ワークフロー **Weekly dashboard** が成功すると、**zip ファイル**として成果物が保存されます。

## 手順（GitHub の画面）

1. リポジトリを開く（例: `ggl81isn-ai/monitoring-production-progress`）。
2. 上部タブの **「Actions」** をクリックする。
3. 左の一覧から **「Weekly dashboard」** を選ぶ。
4. **緑のチェックが付いている実行**（いちばん上が最新）をクリックする。
5. 画面下部付近の **「Artifacts」** セクションを探す。  
   - 見当たらないときは、少し下にスクロールする。
6. **`weekly-dashboard-dist`** という名前のリンクをクリックする。  
   - ブラウザが **zip をダウンロード**する。

## zip の中身

解凍すると、だいたい次のファイルがあります。

| ファイル | 内容 |
|----------|------|
| `index.html` | 生成された週次ダッシュボード（ブラウザで開く） |
| `tailwind.css` | スタイル（`index.html` と同じフォルダに置いて開く） |
| `report.png` | ダッシュボードのスクリーンショット |

**注意:** `index.html` をダブルクリックで開くときは、**解凍したフォルダのまま**にしておき、`tailwind.css` が **同じフォルダ**にある状態にしてください（パスが相対参照のため）。

## うまくいかないとき

- **Artifacts が無い** → その実行は失敗しているか、アップロード前に止まっている。ログで赤いエラーを確認する。
- **zip が落ちない** → ログインしているか、ブラウザのポップアップブロックを確認する。

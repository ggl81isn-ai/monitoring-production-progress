# このプロジェクトで「何が起きているか」（非エンジニア向け）

## 一言で

**Notion（や JSON）のデータを、週次の進捗レポート（Webページ＋画像）に変換する仕組み**です。  
必要に応じて **GitHub で自動実行**し、さらに **LINE に通知**できます。

## 部品の役割（料理に例えると）

| 部品 | たとえ | 何をするか |
|------|--------|------------|
| `report-preview` | レシピと器 | 見た目（デザイン）の元。CSS をビルドすると「器」の形が決まる。 |
| `weekly-dashboard` | キッチン | データを読み、HTML を組み立て、PNG のスクショを撮る。 |
| Notion | 食材置き場 | 本番の進捗データの置き場（使わないならサンプルデータだけでも動く）。 |
| GitHub Actions | 毎週決まった時間の調理人 | サーバー上で同じ手順を自動で繰り返す。 |
| LINE | 配達 | できあがりをスマホに届ける（設定しなければ届かないだけ）。 |

## アカウント・「API」は必須か？

| やりたいこと | アカウントなど | 取得するもののイメージ |
|--------------|----------------|------------------------|
| 自分の PC だけで見た目を試す | **不要**（Git と Node があればよい） | 特になし |
| Notion の本番データで出したい | **Notion**（既に使っているならそのまま） | 「インテグレーション」と **秘密のトークン**、**データベースID** |
| 毎週自動で GitHub 上で作りたい | **GitHub**（リポジトリを置く） | リポジトリの **Secrets**（パスワードのような秘密情報の置き場） |
| LINE に送りたい | **LINE Developers** のチャネル | **チャネルアクセストークン** と **送り先のユーザーID** |

「API を取得する」と書いてあるのは、ざっくり **「コンピュータが Notion や LINE にアクセスするための合言葉（トークン）をコピーして設定する」** という意味です。  
ブラウザで見るだけの Notion 利用とは別に、**自動処理用の許可証**を1つ作るイメージです。

## おすすめの順番（優先度）

1. **ローカルでサンプルだけ通す**（何もアカウント追加なしで「パイプの流れ」がわかる）  
2. 本番データが Notion なら **インテグレーション＋`.env`**  
3. 自動化したければ **GitHub の Secrets と Actions**  
4. 通知まで欲しければ **LINE のトークンとユーザーID**

次のファイルに、画面レベルで「何をクリック・何をコピーするか」まで書きます。
